# 需求文档

## 1. 项目概述

机票价格分析平台，聚合多渠道机票价格数据，提供搜索、历史趋势可视化与智能分析能力，帮助用户找到最佳购票时机。

---

## 2. 功能需求

### 2.1 数据聚合

- 接入 Amadeus API 作为主要全球航班数据源
- 定时任务每日自动抓取并存储目标航线的价格快照
- 支持多货币展示（默认 CNY，可切换）

### 2.2 搜索功能

- 支持按出发地、目的地、出发日期、返回日期（可选）、乘客人数搜索
- 支持机场代码和城市名称两种输入方式，自动补全
- 搜索结果按价格、时长、航班号等字段排序和筛选
- 支持单程 / 往返搜索

### 2.3 价格趋势可视化

- 展示指定航线过去 30 天 / 90 天 / 180 天的价格折线图
- 支持多航空公司价格对比
- 标注历史最低价、平均价、当前价位置
- 支持缩放和时间范围选择

### 2.4 价格智能分析（Agent）

- 用户可用自然语言提问，例如：
  - "下个月北京到上海什么时候最便宜？"
  - "现在买机票合适吗？"
  - "对比一下各平台的价格差异"
- Agent 自动调用搜索、趋势查询、价格分析等工具
- 返回结构化分析报告，包含买入建议和理由

### 2.5 价格提醒（后期功能）

- 用户设置目标价格，价格达到阈值时发送通知（邮件/消息）

---

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 搜索接口响应 < 3s，页面首屏加载 < 2s |
| 可用性 | 服务可用率 > 99%，数据刷新失败不影响前端展示 |
| 安全性 | API Key 不暴露到前端，用户输入做校验防注入 |
| 可维护性 | 数据源模块化，新增数据源只需实现统一接口 |
| 部署 | 支持 Docker 容器化部署，可在云服务器运行 |

---

## 4. 用户故事

| ID | 作为... | 我想要... | 以便... |
|----|---------|-----------|---------|
| US-01 | 用户 | 搜索特定航线的机票价格 | 找到最便宜的航班 |
| US-02 | 用户 | 查看某航线的历史价格趋势图 | 判断当前价格是否合理 |
| US-03 | 用户 | 用自然语言问"什么时候买最合适" | 得到专业分析建议，省去手动对比 |
| US-04 | 用户 | 对比多个航空公司的价格 | 选择性价比最高的航班 |
| US-05 | 用户 | 每天打开网站看到价格汇总 | 随时掌握感兴趣航线的价格动态 |

---

## 5. 边界与限制（不做）

- 不提供机票预订/支付功能，仅做价格展示与分析
- 不做用户账号体系（MVP 阶段）
- 不做移动端 App，仅 Web
- 不做实时价格推送（WebSocket），价格数据为定时快照
