# 开发计划

## 里程碑概览

| 阶段 | 内容 | 预估周期 |
|------|------|----------|
| M1 | 基础搭建 + 数据接入 | 第 1-2 周 |
| M2 | 核心搜索与展示功能 | 第 3-4 周 |
| M3 | 趋势可视化 + 定时快照 | 第 5-6 周 |
| M4 | Agent 智能分析 | 第 7-8 周 |
| M5 | 部署 + 优化 | 第 9-10 周 |

---

## M1：基础搭建 + 数据接入

**目标**：跑通从 Amadeus API 拿到机票数据的完整链路

- [ ] 项目脚手架（后端 FastAPI、前端 React + Rollup）
- [ ] Docker Compose 配置（PostgreSQL + TimescaleDB + Redis）
- [ ] 数据库 Schema 创建，Alembic 迁移初始化
- [ ] Amadeus API 接入，封装 `AmadeusSource`
- [ ] 机场数据导入（IATA 静态数据）
- [ ] 基础健康检查接口 `GET /api/health`

---

## M2：核心搜索与展示功能

**目标**：用户可以搜索机票并看到结果

- [ ] 搜索接口 `POST /api/search` 实现
- [ ] Redis 缓存层接入
- [ ] 前端搜索页面（出发地、目的地、日期、乘客输入）
- [ ] 机场自动补全 `GET /api/airports`
- [ ] 搜索结果列表展示（航班号、价格、时长、排序筛选）
- [ ] Dashboard 首页：热门航线每日价格汇总 `GET /api/dashboard`

---

## M3：趋势可视化 + 定时快照

**目标**：用户可以看到历史价格走势，系统每日自动抓取数据

- [ ] 价格趋势接口 `GET /api/trends`
- [ ] APScheduler 定时任务：每日价格快照抓取
- [ ] 前端趋势图表页面（Recharts 折线图）
- [ ] 价格统计信息展示（历史最低/均价/当前价位）
- [ ] 时间范围选择（30天 / 90天 / 180天）
- [ ] 多航空公司对比图

---

## M4：Agent 智能分析

**目标**：用户可以用自然语言提问获得分析建议

- [ ] Agent 工具定义（search_flights / get_price_trend / analyze_price）
- [ ] Claude API 集成，Tool Use 链路调试
- [ ] 流式输出接口 `POST /api/chat`（SSE）
- [ ] 前端对话框组件，支持流式展示
- [ ] Agent 回复中的结构化数据可视化（如返回图表数据时渲染图表）

---

## M5：部署 + 优化

**目标**：稳定部署到公网，体验流畅

- [ ] Nginx 配置（静态资源 + API 代理）
- [ ] Docker Compose 生产环境配置（环境变量、日志）
- [ ] 性能优化：接口响应时间、前端资源压缩
- [ ] 错误监控与日志收集
- [ ] README 文档完善

---

## 优先级说明

**必须做（MVP）**：M1 + M2 + M3  
**重要**：M4（Agent 是核心差异化功能）  
**后期**：M5 完整部署，价格提醒通知功能
