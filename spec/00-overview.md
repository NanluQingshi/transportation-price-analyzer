# AI Coding 项目规范总览

这套文档已从“前端规范”重组为三层结构：`通用工程规范 + Agent 项目规范 + 前端附录`。

当前目录名仍保留为 `frontend-spec/`，只是为了兼容已有路径；从内容定位上，它已经不再只服务前端项目。

## 如何阅读

- 如果你在做任何 AI Coding 项目：先读 `00`、`01`、`02`、`03`、`05`、`06`
- 如果你在做 Agent / Skill / 工作流项目：重点关注各文档中的“Agent 项目”章节
- 如果你在做带页面或控制台的前端项目：再补读 `04`，以及其他文档中的“前端附录”章节

## 三层结构

### 第一层：通用工程规范

适用于所有 AI Coding 项目，包括 Agent 项目、前端项目、混合项目。

重点解决：

- 代码与文档如何组织
- 任务如何拆分与描述
- 命名、类型、评审、质量卡口如何统一
- 如何把规范文件化，而不是只靠对话传递

### 第二层：Agent 项目规范

适用于 Skill、自动化脚本、工作流编排、工具调用型项目。

重点解决：

- Prompt、规则、模板如何管理
- Tool / adapter 如何分层
- 输出如何结构化并校验
- 回放测试、评测、超时重试、幂等和观测性如何落地

### 第三层：前端附录

只在项目包含页面、控制台、组件层和交互界面时生效。

重点解决：

- React、路由、组件库与样式体系如何选型
- 组件 API、Props、JSDoc、示例如何设计
- 页面结构、UI 测试和交互约束如何统一

## 文件分工

- `01-tech-stack.md`：通用工程基础与技术选型，附 Agent 和前端补充
- `02-structure-and-naming.md`：通用目录与命名规范，附 Agent 结构和前端结构
- `03-coding-and-types.md`：通用编码与类型规范，附 Agent 编排约束和前端补充
- `04-component-and-docs.md`：前端附录，聚焦组件、交互和 UI 文档规范
- `05-style-test-and-quality.md`：通用质量卡口，附 Agent 质量卡口和前端样式/测试补充
- `06-ai-collaboration.md`：通用 AI 协作规范，附 Agent 专项协作要求和前端任务补充

## 使用原则

1. 先用通用规范兜底，再按项目类型叠加专项规范。
2. Agent 项目默认不继承前端约束；只有出现 UI 层时，才启用前端附录。
3. 前端项目如果接入 Agent 能力，也应同时遵守 Agent 章节中的约束。
4. 规范优先写进文件和自动化卡口，不依赖口头记忆。

## 适用建议

- 新项目：从第一层开始建立规范，再补齐专项章节。
- 存量项目：不要求一次性重构，可优先迁移高频改动、风险较高的模块。
- 混合项目：通用规范为底座，Agent 与前端附录并行生效。
