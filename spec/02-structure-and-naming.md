# 通用目录结构与命名规范

## 文档定位

本篇先定义通用目录与命名原则，再分别给出 Agent 项目和前端项目的推荐结构。

## 一、通用原则

### 目录职责

- 目录应按职责而不是按个人习惯组织
- 规则、模板、配置、测试数据、运行代码应显式分层，不混放在同一目录
- 高复用内容放公共目录，业务私有内容就近放置，避免假复用

### 命名规范

- 命名优先表达业务含义，避免 `temp`、`newData`、`handleIt` 这类弱语义命名
- 目录和文件命名保持一致风格，同一层级不要混用多种命名方式
- 对外暴露的模型、类型、配置、输出字段要保持稳定命名，避免下游依赖频繁变更

### 路径规范

- 路径层级不宜过深，减少脆弱的多层相对路径
- 存在别名能力时，优先通过统一别名引用核心目录
- import 顺序保持稳定，交给工具自动排序，不依赖人工维护

## 二、Agent 项目推荐结构

适用于 Skill、自动化脚本、工作流编排、工具调用型项目。

```text
agent-project/
├── docs/                  # 规范、运行说明、协议文档
├── prompts/               # Prompt 模板、规则片段、few-shot 示例
├── schemas/               # 输入输出 schema
├── fixtures/              # 回放数据、样例输入、测试快照
├── src/
│   ├── workflows/         # 主流程编排
│   ├── tools/             # 工具封装层
│   ├── adapters/          # 外部系统适配层
│   ├── parsers/           # 解析逻辑
│   ├── formatters/        # 输出格式化逻辑
│   └── types/             # 类型定义
├── tests/                 # 单测、回放测试、评测
├── scripts/               # setup、debug、smoke test、迁移脚本
└── config/                # 非敏感默认配置
```

### Agent 目录约束

- Prompt、Schema、Fixture 必须独立管理，不与主流程代码混放
- 主流程只做编排；外部系统交互统一经由 `tools` 或 `adapters`
- 解析逻辑与输出格式化逻辑应独立，避免一个文件同时承担输入、调用、解析、输出全部职责
- 运行产物如日志、报告、下载文件应进入临时目录或被忽略目录，不直接写入源码目录

## 三、前端附录

只有项目包含页面和 UI 层时，才参考以下结构。

```text
src/
├── assets/                # 图片、字体等静态资源
├── components/            # 通用 UI 组件
├── pages/                 # 页面模块
│   └── orderList/
│       ├── components/    # 页面私有组件
│       ├── hooks/         # 页面私有 hooks
│       ├── context/       # 页面私有状态上下文
│       └── index.tsx      # 页面入口
├── hooks/                 # 全局通用 hooks
├── services/              # API 请求层
├── stores/                # 全局复杂状态
├── types/                 # 全局类型
├── utils/                 # 工具函数
├── router/                # 路由配置
└── app.tsx
```

### 前端补充约束

- `pages/` 下页面目录统一使用 camelCase
- `components/` 下组件目录统一使用 PascalCase
- 通用 hooks 统一以 `use` 开头
- 页面私有组件不进入全局 `components/`，避免误复用
- 单个组件文件建议不超过 300 行，复杂逻辑优先拆 hooks 或子组件

## 四、统一原则

1. 先用通用原则做分层，再决定采用 Agent 结构还是前端结构。
2. 同一仓库可以同时存在 Agent 模块和前端模块，但各自的目录约束必须清晰。
3. 不要为了追求“看起来统一”，把不同职责的文件硬塞进同一种结构。
